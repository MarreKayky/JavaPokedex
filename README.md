# 🎮 JavaPokedex - Simulação de Back-end Pokémon

O **JavaPokedex** é um projeto de back-end desenvolvido para simular a lógica central de um jogo de Pokémon. O objetivo principal foi colocar em prática conceitos fundamentais de Java, explorando estruturas de dados, integração com APIs externas e lógica de gerenciamento de inventário.

---

## 🚀 Sobre o Projeto

Este projeto nasceu da vontade de unir o aprendizado de programação com uma paixão pessoal por cultura pop e jogos. A ideia foi construir uma base sólida para o funcionamento de um sistema Pokémon, focando inteiramente na inteligência e nas regras de negócio do back-end.

Durante o desenvolvimento, utilizei o **Google Gemini** como um "mentor digital" para entender requisitos técnicos mais complexos. Essa colaboração me permitiu aprender e aplicar em tempo real técnicas que eu ainda não dominava, como o consumo de APIs REST e manipulação avançada de listas.

---

## 🛠️ Funcionalidades Principais

* **Exploração e Captura:** Sistema de encontro aleatório com Pokémon via integração real com a **PokéAPI**.
* **Mochila de Itens:** Gerenciamento de diferentes tipos de Pokébolas (Comum, Great, Ultra e Master) com multiplicadores de chance de captura.
* **Pokédex Detalhada:** Registro histórico de capturas contendo Tipo, HP, Bioma, Descrição oficial e Data/Hora exata da captura.
* **Sistema de Recompensas:** Mecânica de envio de Pokémon para o **Professor Carvalho** em troca de novos itens, com lógica de sorteio (RNG) para prêmios.
* **Seleção Múltipla:** Possibilidade de gerenciar e enviar vários Pokémon simultaneamente para otimizar a experiência do jogador.

---

## 💻 Tecnologias Utilizadas

* **Java (JDK 15+)** - Linguagem base do projeto.
* **HttpClient** - Para requisições assíncronas à PokéAPI.
* **Gson** - Biblioteca para conversão e tratamento de dados JSON.
* **Git & GitHub** - Controle de versão e documentação.
